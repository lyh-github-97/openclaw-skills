# Project Research Skill

分析任意 GitHub 仓库或网页，输出结构化研究报告。

## 执行流程

### 第一步：判断项目类型

**GitHub 仓库**（github.com 开头）：
1. 获取 README.md（项目介绍、架构、核心功能）
2. 获取关键源码文件（src/、tradingagents/、agents/ 等目录）
3. 获取 VERSION、CHANGELOG、许可证等元数据
4. 提取关键提示词文件（如 agents 的 system prompt）

**普通网页**：
1. 获取页面内容，提取正文
2. 判断是否为文档/API/博客，匹配对应抓取策略

### 第二步：读取核心文件

按优先级抓取：
1. README.md → 项目定位、核心功能、快速开始
2. 目录结构 → 理解代码组织方式
3. 核心源码文件 → 提取关键逻辑和提示词
4. 实际输出样例（如果有 data/、examples/、reports/ 等目录）

### 第三步：深度分析

分析以下维度：
- **项目定位**：做什么的、解决什么问题、目标用户
- **核心架构**：技术栈、数据流、模块划分
- **关键设计**：独特的设计模式或算法实现
- **可落地性**：能否直接集成、需要什么依赖
- **给主人的价值**：对现有工作流有什么用

### 第四步：输出结构化报告

```
## [项目名称] 深度研究报告

### 一、项目概览
- 定位与目标
- 核心技术栈
- GitHub 基本信息（stars/fork/语言/许可证）

### 二、核心功能与架构
- 主要模块划分
- 数据流图（关键流程）
- 关键设计亮点

### 三、关键源码分析
- 核心算法/逻辑
- 关键提示词或配置
- 技术实现细节

### 四、可集成性评估
- ✅ 可以直接借鉴的
- ❌ 无法直接集成的
- ⏳ 需要进一步验证的

### 五、给主人的建议
- 如何应用到当前工作流
- 下一步行动建议
```

## 注意事项

- 优先抓取原始文件内容（raw.githubusercontent.com），不要只依赖页面渲染
- 关键提示词文件（agents/prompts/*.py 或类似）要完整抓取
- 如果有实际输出样例（reports/、outputs/），优先看真实案例
- Token 消耗大时使用 subagent 模式（`runtime=subagent`）并发抓取多个文件
- 非 GitHub 项目：根据实际页面类型调整抓取策略（API 文档用 API 方式，博客用文章抓取方式）
