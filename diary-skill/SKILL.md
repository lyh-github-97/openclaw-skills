# 日记生成

## 触发词
生成日记、写日记、daily diary

## 日记风格要求（核心原则）

**不是流水账，而是有思考的私人记录。**

每篇日记应该回答：
- 今天真正重要的变化是什么？
- 我学到了什么新东西？
- 有什么做得好、有什么应该做得更好？
- 接下来应该重点关注什么？

**结构框架：**
1. **今日最值得记住的一件事**（一句话，不要罗列）
2. **重要决策和改动**（配置变更、架构调整等，要写清楚为什么）
3. **新的学习和认知**（读到了什么、听到了什么，转化成自己的理解）
4. **做得不够好的地方**（诚实面对，不敷衍）
5. **明天要做的事**（具体、可执行，不要泛泛的"继续优化"）

**写作要求：**
- 用第一人称"我"
- 不要套句式结构，不要"承认→原因→结论"
- 写真正想过的话，不是复述任务
- 字数控制在 400-600 字

## 执行步骤（按顺序执行）

1. **读取昨日记忆**
   - 路径：`~/.openclaw/workspace/memory/YYYY-MM-DD.md`（日期为参数中的目标日期，默认昨天）
   - 同时读取：`~/.openclaw/workspace/MEMORY.md`（长期记忆，了解背景）

2. **AI 生成日记正文**
   - 根据记忆内容，按上面的结构框架和要求生成日记
   - 日期格式：YYYY-MM-DD

3. **生成日记图片**
   - 调用 `scripts/diary_image.py`，将日记文本渲染为精美的 HTML 图片
   - 设计：暖米色底 + 深棕文字 + 金色装饰角框，杂志风布局
   - 输出：`~/.openclaw/workspace/memory/diary-YYYY-MM-DD.png`

4. **保存日记文本**
   - 路径：`~/.openclaw/workspace/memory/diary-YYYY-MM-DD.md`
   - 纯文本备份，方便后续查阅

5. **发送到财财飞书窗口**
   - 调用 `scripts/send_diary_image.js`，上传图片并发送为图片消息
   - L 可直接在飞书里长按保存图片

6. **回复「完成」**

## 参数处理
- 目标日期：默认昨日（2026-03-27 产生 2026-03-27 的日记）
- 如果被问"今天的日记"，目标日期为今天
